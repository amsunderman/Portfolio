Project Description

For Computer Science 229 (Advanced Coding Practices), Our first project was the implementation of a meme generator. It would take in an ACT, MEM, and FSF files and use the data contained within them to generate a meme. The FSF file essentially defined the font to use for the meme. It contained an image of the font as well as the coordinates, width, and height of each letter. The MEM file stored data about the images to be used for memes. It contained the images themselves as well as coordinates to locate meme text at. The coordinates were assigned to named destinations, for example, coordinates near the top might simply be labeled TOP. The ACT file stored “actions”. An action would detail which font to use, which image to use it on, which location to put it at, and the actual text to write.

Design

        To complete this project, I broke the overall problem down into small and manageable steps. The images we were taking in were in .simp format (custom image format for this project). I decided to start by making a simp_util source and header file to define all interactions with simp files. I created a structure that stored all relevant data to a simp file and then used the functions in this source file to modify the data contained in these structures in useful ways (crop, black/white, colorshifts, and overlay). This source file also contained the code to write new images back into the .simp file format. From here, I created a meme_file_util source and header that defined all interaction with MEM, ACT, and FSF files. Its main two functionalities are to read in the three file types and store them into relevant structures and then use these structures to execute each action in the ACT file. I then wrote a source file that would eventually be the executable (contained the main method) that took in the command line arguments, parsed them, and then used simp_util and meme_file_util to complete the necessary work to generate the meme.
