Project Description

While employed at Huffman Engineering, they were looking for an automated solution to controlling revision data for sets of drawings. Currently, they manually went through and edited the revision data for each title block. It was essential that their solution would be implemented in a way that users with both AutoCAD and AutoCAD Lite could interact and modify the data. It was also important that the solution allow for a revision to be added to all drawings simultaneously or a specific drawing.

Design

        I designed a system using a macro-enabled excel spreadsheet. Through the use of VBA, I created a spreadsheet that could be saved in the same folder as the drawing files for the current set. The user could initialize it, import data from AutoCAD, add a revision to all drawings, add a revision to (a) specific drawing(s), export data to AutoCAD, and make use of several convenient key bindings to speed up data entry. When initialized, the system would make blank revision tables for all .dwg files in the current directory it was saved in. The system interacted with AutoCAD objects through built-in VBA functionality, and was able to export the data from AutoCAD via an algorithm that searched all .dwg files for the title block headers needed to populate the excel sheet. The spreadsheet had a build in revision line that was used to add to all (implementing a F.I.F.O. queue if a revision table was full). The user could also use Excel’s built in find functionality to find a specific table to add a revision into. With the same algorithm as exporting, the system was also able to push the data into AutoCAD. I was also able to speed up data entry by allowing the user to put in the revision number followed by the system auto-populating other fields via the company revision policy.

        The Excel sheet was also very robust and would detect the user’s operating system, excel version, and AutoCAD version. Upon analyzing this data, the system would inform the user what actions they could and could not perform. AutoCAD LT users could not use the import and export functions since Light doesn’t have any VBA functionality, however, when they attempted to perform those actions, the system would email a user with full AutoCAD with a link to the spreadsheet so it could be properly updated.
